## 3. 训练

烟测：

```bash
python -m gap_step.train --config gap_step/configs/train_teacher_smoke.yaml
```

完整训练：

```bash
python -m gap_step.train --config gap_step/configs/train_teacher_full.yaml
```

C5 快速调参：

```bash
python -m gap_step.train --config gap_step/configs/train_teacher_c5_tune.yaml
```

说明：`train_teacher_c5_tune.yaml` 是诊断 C5 的放宽口径配置，当前使用 `robot_radius=0.1`、`safe_margin=0.0`、`max_steps=800`。它不等同于 full 配置。

训练顺序：

```text
C1 -> C1_5 -> C2A -> C2B -> C3 -> C4 -> C5
```

训练规则：

- 后一课程继承前一课程模型参数
- 每个课程重置优化器
- 每个课程只保存最终模型
- 不保存 `teacher_best.pt`
- 终端输出中文实时指标，不显示进度条
- PPO 使用显式旧策略：`model_old` 采样，`model` 更新，更新后同步

日志示例：

```text
课程 C1 | 更新 12 | 阶段步数 1536 | 回合 8 | 成功率 0.125 | 滚动成功率 0.100 | 碰撞率 0.750 | 超时率 0.125 | 平均回报 -8.42 | 动作范数 2.10 | 熵 2.91 | KL 0.0120 | 裁剪率 0.125 | 解释方差 0.500 | PPO更新 4
```

## 5. 评估

评估最终课程模型：

```bash
python -m gap_step.evaluate --checkpoint checkpoints/C5/teacher_final.pt
```

评估 C5 tune：

```bash
python -m gap_step.evaluate --checkpoint checkpoints/C5/teacher_final.pt --episodes 50 --stages C5 --output results/eval_c5_tune.csv
```

只评估 C1：

```bash
python -m gap_step.evaluate --checkpoint checkpoints/C1/teacher_final.pt --episodes 20 --stages C1
```

用 C5 模型做全课程评估：

```bash
python -m gap_step.evaluate --checkpoint checkpoints/C5/teacher_final.pt --episodes 20 --stages C1,C1_5,C2A,C2B,C3,C4,C5
```

默认不传 `--stages` 时会跑 C5 的 `id_test`、`ood_size_test`、`ood_dynamics_test` 三个 split。

## 6. 可视化

```bash
python -m gap_step.visualize --checkpoint checkpoints/C5/teacher_final.pt
```

GIF 输出到 `results/`。

可视化会复用 checkpoint 中保存的环境配置，确保和训练/评估口径一致。

当前默认输出：

```text
results/typical_success.gif
results/typical_wait.gif
results/typical_collision.gif
```

注意：文件名是固定案例名；当前 checkpoint 下三个默认 seed 都成功，`typical_collision.gif` 不一定代表本轮碰撞。

## 7. 当前重点

当前 C5 tune 正式 50 回合 ID 评估最好为 `0.68`，尚未真实超过 `0.70`。主要失败类型是 closed gate collision，其次是 wall collision。

50 回合泛化评估结果：

```text
id_test          0.68
ood_size_test    0.64
ood_dynamics     0.40
```

继续调参时优先看：

- success rate
- collision rate
- timeout rate
- entropy
- std
- approximate KL
- clip fraction
- explained variance
- PPO 更新次数

## 8. 当前训练稳定性设置

```yaml
learning_rate: 0.0001
target_kl: 0.2
entropy_coef: 0.0001
log_std_init: -1.0
min_log_std: -2.0
max_log_std: 0.0
```

环境奖励设置中启用：

```yaml
reward_progress: 4.0
reward_timeout: -20.0
suppress_positive_progress_on_collision: true
```

含义：动态几何 progress 是主要 shaping；超时和碰撞一样是明确失败；撞墙或撞门时，不允许拿正的 progress reward。
