# GAP-Step v4.6 实验操作手册

本文档对应当前重构后的版本：连续二维旋转时变窗口迷宫 PPO 特权教师训练。当前不再运行视觉学生、BC、Aux、启发式教师演示、`trainers/` 或 `scripts/`。

## 1. 环境准备

进入项目：

```bash
cd /home/jack/wyh/时变窗口
conda activate wyh
```

检查测试：

```bash
pytest -q
```

当前应看到：

```text
15 passed
```

## 2. 当前代码入口

所有代码入口都在 `gap_step/`：

```text
python -m gap_step.train
python -m gap_step.evaluate
python -m gap_step.visualize
```

旧版演示生成、行为克隆、视觉学生训练和旧可视化命令已从当前代码中移除。

## 3. 观测约定

特权教师输入固定为：

```text
obs_dim = 4 + 3 + 32 = 39
```

射线设置：

```text
N_ray = 32
ray_max_dist = 0.35 * S
```

其中 `S` 是当前 episode 实际采样到的连续迷宫边长。例如：

```text
S=15 -> ray_max_dist=5.25
S=31 -> ray_max_dist=10.85
```

## 4. 烟测训练

当前迷宫不是固定墙体模板，而是每个 episode 在线生成普通网格迷宫拓扑，再转换成连续墙段和时变窗口。代码保持在 `gap_step/` 内，核心几何和窗口安全判定处保留了简短中文注释。

先运行短训练，确认代码、环境和 checkpoint 保存都正常：

```bash
python -m gap_step.train --config gap_step/configs/train_teacher_smoke.yaml
```

预期输出：

```text
checkpoints/teacher_final.pt
results/train_metrics.csv
```

注意：smoke 配置只用于检查流程，不代表策略已经收敛。

## 5. 完整训练

完整课程训练：

```bash
python -m gap_step.train --config gap_step/configs/train_teacher_full.yaml
```

课程顺序固定：

```text
C1 -> C2 -> C3 -> C4 -> C5
```

默认完整配置：

```text
steps_per_stage = 1,000,000
total_steps = 5,000,000
```

当前训练配置启用连续几何动态进展 shaping，以避免稀疏奖励下策略学成“等待超时”：

```text
reward_progress = 2.0
reward_timeout = -5.0
progress_mode = dynamic_geometry
gate_lookahead_time = 20.0
```

该 shaping 只影响训练奖励，不进入教师观测。它基于连续几何 visibility roadmap 估计到目标的剩余时间，并把窗口未来是否可通行及等待代价纳入路径势函数。环境默认仍保留 strict v4.6 稀疏奖励；需要复现严格稀疏奖励时，将 `reward_progress` 设为 `0.0` 或移除该配置项。

## 6. 评估

评估最终教师：

```bash
python -m gap_step.evaluate --checkpoint checkpoints/teacher_final.pt
```

快速评估可减少 episode 数：

```bash
python -m gap_step.evaluate --checkpoint checkpoints/teacher_final.pt --episodes 5
```

评估输出：

```text
results/eval_metrics.csv
```

评估 split：

```text
id_test            seeds 10000..10199
ood_size_test      seeds 20000..20199
ood_dynamics_test  seeds 30000..30199
```

## 7. 可视化

生成三类 GIF：

```bash
python -m gap_step.visualize --checkpoint checkpoints/teacher_final.pt
```

输出：

```text
results/typical_success.gif
results/typical_wait.gif
results/typical_collision.gif
```

当前可视化使用固定 seed 生成典型 rollout。后续可以改成自动搜索真实成功、等待和碰撞样例。
